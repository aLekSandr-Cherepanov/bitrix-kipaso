README: Шаблон компонента dresscode:catalog.item (detail)

Описание
Эта папка (/local/components/dresscode/catalog.item/templates/detail) содержит кастомный шаблон компонента dresscode:catalog.item для отображения детальной страницы товара в интернет-магазине на платформе 1C-Bitrix.
Шаблон был скопирован из /bitrix/components/dresscode/catalog.item/templates/detail в /local для безопасного редактирования без риска перезаписи при обновлении ядра Bitrix. Папка /local имеет приоритет над /bitrix, поэтому система автоматически использует этот шаблон.

Структура
template.php: Основной файл шаблона, содержащий HTML и PHP-код для отображения детальной страницы товара.
style.css: Стили CSS для оформления страницы.
script.js: JavaScript-код для интерактивных элементов (если применимо).
(другие файлы): Могут включать дополнительные ресурсы, такие как изображения, шрифты или файлы локализации.

Назначение
Шаблон используется для:
Отображения информации о товаре (название, цена, описание, изображения и т.д.).
Реализации кастомной логики и стилей, специфичных для проекта.
Интеграции с другими компонентами Bitrix, такими как корзина или каталог.
Инструкции по редактированию

Редактирование:

Вносите изменения в файлы в этой папке (template.php, style.css, script.js и др.).
Не редактируйте аналогичные файлы в /bitrix, так как они не используются и могут быть перезаписаны при обновлении системы.
Очистка кеша:
После внесения изменений очищайте кеш Bitrix, чтобы обновления применились:
В админке: Настройки → Инструменты → Очистка кеша.
Через консоль: php /path/to/bitrix/.settings.php cache clear.

Права доступа:

Убедитесь, что файлы имеют права 644, а папки — 755.
Владелец файлов должен соответствовать пользователю веб-сервера (например, www-data).

Резервная копия:

Перед внесением изменений создайте резервную копию папки, например:
cp -r /local/components/dresscode/catalog.item/templates/detail /local/components/dresscode/catalog.item/templates/detail_backup

Примечания

Если компонент перестаёт работать, проверьте:
Корректность структуры папок в /local/components/dresscode/catalog.item.
Наличие всех необходимых файлов компонента (например, component.php, .description.php).
Логи ошибок Bitrix (/bitrix/logs/) или веб-сервера.
Для диагностики можно временно переименовать исходную папку в /bitrix (например, в detail_backup), чтобы убедиться, что система использует /local.

История

Дата создания: Май 2025

Причина переноса: Перенос шаблона в /local для кастомизации детальной страницы товара без риска потери изменений при обновлении Bitrix.
